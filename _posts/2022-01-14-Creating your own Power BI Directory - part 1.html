In Power BI, reports and dashboards are distributed through apps and workspaces, they serve the purpose, but not an elegant solution.

If you work for a large organization, you would have come across apps for different departments. What if you wanted a Power BI directory for your organization?

Before we begin, first, we need all report names and URLs, this is where Power BI REST APIs will help us

You will find all REST APIs related to Power BI here - Power BI REST APIs - Power BI REST API | Microsoft Docs

To get the data we need, I used "Apps - Get Apps" and "Reports - Get Reports" APIs

Power BI REST API page Get Apps result

Use the JSON connector within Power BI to import both the files, create a relationship between them, the key column in both the data sources will be your "app id" column,

Use a matrix visual to show to create simple list like below, you can also use the conditional formatting option to enable URL for all the reports.

Model view URL conditional formatting option Simple view of the data we collected

You can also use your PowerPoint skills to create the view and import the image as canvas background, this way you can build a nice report directory.

In an another blog, I'll explain how I use PowerPoint to create my layouts, before starting development in Power BI Desktop.

Final step in the solution we built is to automate the API calls, which is very important, because report URLs might change or there will be new report to add.

For this, we can use Power Automate to run the API calls, store the data in OneDrive excel, trigger Power BI dataset refresh, I'll cover this in another blog.
